AUTHORITATIVE SIMULATOR ARCHITECTURE OVERRIDE

This section overrides any older contradictory guidance in this file.

This override applies to every simulator execution mode and entrypoint, including initial run, cold start, cold build, cache miss rebuild, allow_rebuild, refresh, explicit recalc, admin canonical recalc, user-triggered rebuild, artifact refresh, artifact ensure, snapshot-producing rebuilds, and any future renamed equivalent. No execution mode is exempt.

User Past Sim and GapFill may start with different inputs, but after input normalization actual-house and test-house GapFill runs must enter the exact same shared simulator producer path as the normal user flow.

There is only one pre-DB simulator producer path.

Producer labels must include:
- TRAVEL_VACANT
- TEST

Fingerprint rule:
- exclude only TRAVEL_VACANT days from the usage fingerprint
- keep TEST-day actual usage included in the usage fingerprint

After simulated day outputs are derived and stored:
- stitch uses ACTUAL + TRAVEL_VACANT simulated days only
- compare uses TEST simulated days only against actual interval data for those same days

TEST simulated days do not go into the stitched Past chart.

Implementation detail (must stay aligned with code): the shared producer may emit pre-projection `SIMULATED_TEST_DAY` tags on daily/intervals for canonical totals; **user-facing** Past applies **`projectBaselineFromCanonicalDataset`** so validation dates display as **ACTUAL (VALIDATION/TEST)** with meter kWh, and scales **15m intervals** on those dates to the meter day total. **`rehydrateValidationCompareMetaFromBuildInputsForRead`** restores `validationOnlyDateKeysLocal` when cache JSON dropped keys so projection + compare attach both run. TRAVEL_VACANT stays **SIMULATED (TRAVEL/VACANT)** in stitch.
- **Past Sim compare UX:** On the Usage Past tab, **Validation / Test Day Compare** is **collapsed by default**; use **Show details** to expand the metrics + table. Compare rows may include **day-level weather** (Avg/Min/Max °F, HDD65, CDD65) copied from **`dataset.dailyWeather`** for the same local date when the Past dataset has that map—**read-only context**, not a second simulator or scoring path; missing dates show as missing (no fake fill).
- On cache restore, **`reconcileRestoredPastDatasetFromDecodedIntervals`** (see `modules/usageSimulator/dataset.ts`) recomputes `daily` and `series.daily` from decoded **`intervals15`** and applies simulated-day ownership from persisted **meta** when explicit fields exist, so old plain `SIMULATED` rows cannot survive as display truth for a current run. Legacy artifacts without those meta fields still use stored `daily` for membership and may copy TRAVEL/TEST `sourceDetail` from the pre-reconcile row when meta has no per-day map entry. **`meta.simulatedSourceDetailByDate`** may also record **`SIMULATED_INCOMPLETE_METER`** / **`SIMULATED_LEADING_MISSING`** for stitch fills. After **`projectBaselineFromCanonicalDataset`**, Past chart and daily table both read the same projected **`daily`/`series.daily`** (Usage fallback from **`series.daily`** preserves **`source`/`sourceDetail`**).

AUTHORITATIVE FAIL-CLOSED TRUTH OVERRIDE

This section overrides any older contradictory guidance in this file.

Truth-preserving fallbacks are allowed. Truth-substituting fallbacks are not.

Allowed examples:
- cached weather in DB falling back to a fresh weather API for the same weather truth
- primary weather provider falling back to a backup weather provider when the backup still returns correct weather truth for the same request
- equivalent trusted source-layer fallback that preserves the same identity and same truth

Forbidden fallbacks include anything that can return bad, stale, mixed, substitute, guessed, zero-filled, partial, or misleading truth.

If the correct simulator truth, compare truth, artifact truth, or stored-output truth cannot be proven correct, the system must fail with an explicit failure status, code, and message.

When exact identity is required, failure to resolve exact identity must not fall back to latest artifact, latest by scenario, compatible identity, or any substitute source.

Compare truth must remain stored-output-backed or artifact-backed. Missing compare simulator totals must not be converted into numeric 0 values and shown as valid compare output.

Fresh diagnostics are diagnostics only. They must not replace compare truth. If canonical fresh scored day totals are required and missing, diagnostics must fail explicitly.

Whenever a new simulator run produces new simulated outputs for the same scope and identity, prior simulated outputs must be cleared, replaced, or fully overwritten so stale outputs are not mixed into the current result set.

If a fallback preserves the same truth, it is allowed. If it can produce bad data or misleading data, it is forbidden.
